# 開発規則・ガイドライン

このディレクトリには、紙アンケート OCR 入力効率化 Web アプリの開発で使用する共通の規則とガイドラインを格納しています。

## ファイル一覧

### 1. [code-quality-standards.md](./code-quality-standards.md)

コード品質と保守性に関する基準

**主要内容:**

- テスト駆動開発（TDD）ガイドライン
- コード品質指標と基準
- 保守性を向上させるプラクティス
- プロジェクト段階別の品質バランス
- ボーイスカウトルールの適用

### 2. [language-rules.md](./language-rules.md)

プロジェクト内での言語使用規則

**主要内容:**

- コード内コメント: 英語
- ドキュメントファイル: 日本語
- コミットメッセージ: 英語（コンベンショナルコミット）
- 関数・変数名: 英語
- エラーメッセージ: ユーザー向け=日本語、ログ=英語

### 3. [ai-documentation.md](./ai-documentation.md)

ドキュメント駆動開発のワークフローと規則

**主要内容:**

- 5 種類のドキュメント構造（PROMPT → ISSUE → RESEARCH → PLAN → SPEC+TEST → LOG）
- 各ドキュメントの作成タイミングと更新ルール
- ファイル命名規則
- AI 協働開発の最適化手法

### 4. [dependency-mapping.md](./dependency-mapping.md)

ファイル間の依存関係管理規則

**主要内容:**

- DEPENDENCY MAP の記載方法
- Parents（このファイルを使用するファイル）の特定
- Dependencies（このファイルが使用するファイル）の管理
- 影響範囲の分析手法

## 適用プロジェクト

**対象**: 紙アンケート OCR 入力効率化 Web アプリ
**技術スタック**: React.js + TypeScript + Tesseract.js
**開発手法**: ドキュメント駆動開発 + テスト駆動開発

## 規則の更新

開発規則は、プロジェクトの進行に合わせて適宜更新されます。

### 更新手順

1. 規則変更の提案 → Issue として記録
2. チーム内での議論・合意
3. 規則ファイルの更新
4. 既存コードへの影響評価・対応

### 更新履歴

- 2024-11-02: 初版作成（プロジェクト開始時）

## 規則の適用

これらの規則は、以下の場面で参照・適用されます：

- **開発開始時**: プロジェクト構造の設計
- **コード作成時**: 命名規則・品質基準の適用
- **レビュー時**: 品質チェック・規則遵守の確認
- **リファクタリング時**: 依存関係分析・影響範囲の特定
- **ドキュメント作成時**: 構造・命名規則の適用

詳細な適用方法については、各規則ファイルを参照してください。
